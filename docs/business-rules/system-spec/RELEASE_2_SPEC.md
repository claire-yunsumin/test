# Release 2 Spec

## 목표

기본 CRUD 위에 협업 품질을 올립니다. 핵심은 노트 참조, 멘션 정합성, 우측 스레드/타임라인, Inbox 라우팅입니다.

## 범위

- `referencedNoteIds` 가시성 검증
- `MEMBER`, `TASK`, `FORM_FIELD`, `NOTE` 멘션 검증
- 커맨드 검색형 `@`/`#` 작성 UX
- 댓글 작성/수정 이벤트
- 상태 전이와 타임라인 기록
- Inbox 라우팅

## 시스템 요구사항

1. 가시 범위 밖 노트 참조는 `INVALID_NOTE_REFERENCE`로 실패합니다.
2. 유효하지 않은 멘션 대상은 `INVALID_MENTION`으로 실패합니다.
3. `FORM_FIELD` 멘션은 대상 태스크의 템플릿 필드 존재를 확인합니다.
4. 우측 패널은 스레드와 타임라인 탭으로 전환됩니다.
5. 전이 요청은 `reason`을 필수로 받습니다.
6. 댓글/멘션 이벤트는 Inbox와 engagement event에 반영됩니다.
7. 전이 이벤트는 Timeline, Inbox, engagement event에 반영됩니다.

## 수용 기준

- 보이지 않는 태스크의 노트를 댓글에 참조할 수 없습니다.
- `@`/`#` 입력 시 검색 메뉴로 후보를 선택하고 본문 토큰으로 삽입됩니다.
- 댓글의 사람/노드/필드/노트 멘션이 서버 검증을 통과해야 저장됩니다.
- 우측 타임라인 탭에서 상태 전이와 노트 이벤트 흐름을 확인할 수 있습니다.
