# Release 2 Spec

## 목표

기본 CRUD 위에 협업 품질을 올립니다. 핵심은 노트 참조, 멘션 정합성, 우측 스레드/타임라인, Inbox 라우팅입니다.

## User Story (디자인 기준)

- 기획자로서, 나는 스레드에서 `@`/`#` 검색으로 정확한 사람/노트를 빠르게 참조하고 싶다. 그래서 의사결정 맥락이 누락되지 않는다.
- 작성자로서, 나는 노트를 태그로 분류하고 링크를 공유해 논의를 연결하고 싶다. 그래서 제안/결정 상태를 팀이 즉시 이해한다.
- 리뷰어로서, 나는 타임라인에서 상태 전이와 논의 흐름을 이어서 보고 싶다. 그래서 변경 이유를 추적할 수 있다.

## Design Scope (UI/상태/피드백)

- 우측 패널은 `논의`/`변경 기록` 탭을 유지하고, 탭 전환 상태를 예측 가능하게 보존합니다.
- 스레드 입력은 자유 타이핑 + 커맨드 검색(`@`, `#`) 혼합 흐름을 방해하지 않아야 합니다.
- 노트 카드(접힘 상태)는 제목, 요약, 작성자/시간, 태그를 한눈에 보여주는 정보를 우선 배치합니다.
- 노트 링크 복사 및 붙여넣기 시 참조 토큰 변환 여부를 사용자에게 명확히 피드백합니다.

## Do / Don't

### Do

- 멘션 후보 선택 전/후 상태를 시각적으로 구분합니다.
- 노트 태그는 읽기/편집 모두에서 동일한 의미 체계로 노출합니다.
- 노트 참조 실패(`INVALID_NOTE_REFERENCE`)와 멘션 실패(`INVALID_MENTION`)를 구분해 안내합니다.

### Don't

- `@`/`#` 입력 흐름을 모달 강제 방식으로 끊지 않습니다.
- 노트 태그를 접힘/펼침 상태에서 중복 노출해 정보 소음을 만들지 않습니다.
- 스레드와 타임라인의 이벤트 용어를 불일치하게 쓰지 않습니다.

## CRUD 보완 체크 (디자인/기획 공통)

- Create: 댓글/노트 생성 시 참조 대상(`@`, `#`) 유효성을 저장 전에 확인 가능한 피드백으로 제공합니다.
- Read: 논의/타임라인은 같은 맥락을 다른 시점으로 보여주되 이벤트 용어와 라벨을 통일합니다.
- Update: 노트/댓글 수정 시 기존 참조 무결성을 재검증하고 실패 항목을 명확히 표시합니다.
- Delete: 노트/댓글 삭제 시 참조 끊김 영향(스레드 문맥, 타임라인 기록 유지 여부)을 안내합니다.

## 엣지케이스 / 예외사항

- 보이지 않는 노트를 붙여넣어 참조한 경우: `INVALID_NOTE_REFERENCE` 처리 + 대체 행동 안내
- 멘션 대상이 삭제/권한변경된 경우: `INVALID_MENTION` 처리 + 재선택 유도
- 노트 제목/본문은 비었는데 태그만 있는 경우: 저장 허용 정책 여부를 명시하고 UI 일관 유지
- 삭제된 노트를 과거 스레드가 참조하는 경우: 참조 배지의 상태(삭제됨/접근불가) 표시 규칙 정의

## 범위

- `referencedNoteIds` 가시성 검증
- `MEMBER`, `TASK`, `FORM_FIELD`, `NOTE` 멘션 검증
- 커맨드 검색형 `@`/`#` 작성 UX
- 노트 태그(`tags`) 생성/수정/조회
- 노트 링크 공유와 스레드 `#노트` 참조 변환 UX
- 댓글 작성/수정 이벤트
- 상태 전이와 타임라인 기록
- Inbox 라우팅

## 시스템 요구사항

1. 가시 범위 밖 노트 참조는 `INVALID_NOTE_REFERENCE`로 실패합니다.
2. 유효하지 않은 멘션 대상은 `INVALID_MENTION`으로 실패합니다.
3. `FORM_FIELD` 멘션은 대상 태스크의 템플릿 필드 존재를 확인합니다.
4. 우측 패널은 스레드와 타임라인 탭으로 전환됩니다.
5. 전이 요청은 `reason`을 필수로 받습니다.
6. 댓글/멘션 이벤트는 Inbox와 engagement event에 반영됩니다.
7. 전이 이벤트는 Timeline, Inbox, engagement event에 반영됩니다.
8. 노트 생성/수정은 `title`, `content`, `tags`를 지원해야 합니다.
9. 노트 링크를 스레드 composer에 붙여넣으면 `#노트` 참조 토큰으로 치환될 수 있어야 합니다.

## 수용 기준

- 보이지 않는 태스크의 노트를 댓글에 참조할 수 없습니다.
- `@`/`#` 입력 시 검색 메뉴로 후보를 선택하고 본문 토큰으로 삽입됩니다.
- 댓글의 사람/노드/필드/노트 멘션이 서버 검증을 통과해야 저장됩니다.
- 우측 타임라인 탭에서 상태 전이와 노트 이벤트 흐름을 확인할 수 있습니다.
- 노트 태그가 저장/수정되며 접힌 노트 헤더에서 분류 라벨로 확인할 수 있습니다.

## 수용 기준 (User POV)

- 사용자는 노트 링크를 복사해 스레드에 붙여넣을 때 `#노트` 참조로 자동 연결되는 것을 확인할 수 있습니다.
- 잘못된 멘션/참조를 넣고 저장하면 어떤 항목이 문제인지 즉시 알 수 있습니다.
- 논의 탭과 변경 기록 탭을 오가도 맥락이 끊기지 않고 동일 태스크 흐름으로 이해됩니다.
