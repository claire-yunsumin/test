# Release 3 Spec

## 목표

Objective 1/KR 1.1의 "형상화 -> 정형화 -> 멘션" 루프를 태스크 운영 화면 안에서 구현합니다.

## User Story (디자인 기준)

- PM으로서, 나는 자유 노드에서 빠르게 초안을 만들고 필요할 때 템플릿으로 정형화하고 싶다. 그래서 탐색 속도와 운영 일관성을 둘 다 확보한다.
- 작성자로서, 나는 설명/노트를 리치에디터로 작성하고 이미지를 붙여넣어 근거를 남기고 싶다. 그래서 문서화 마찰을 줄인다.
- 협업자로서, 나는 우측 상단 미니맵으로 이 태스크의 상하위 맥락을 즉시 파악하고 싶다. 그래서 판단 시간을 줄인다.

## Design Scope (UI/상태/피드백)

- FREEFORM과 TEMPLATED의 편집 경험 차이를 화면에서 명확히 구분합니다.
- `__task_description`과 노트 본문은 동일한 리치에디터 패턴(툴바, 미리보기, 붙여넣기 업로드)을 사용합니다.
- FREEFORM Form Output은 블록 추가 중심 UX로 구성하고, 블록 타입 선택 메뉴 접근성을 보장합니다.
- 레거시 파일 필드 제거는 사용자에게 "첨부는 Attachments 섹션에서 관리"라는 규칙으로 일관 안내합니다.
- 우측 미니맵은 head/tail 존재를 한눈에 보여주고 노드 클릭 이동을 지원합니다.

## Do / Don't

### Do

- FREEFORM은 빠른 구성, TEMPLATED는 필드 일관성을 강조하는 UI 톤으로 분리합니다.
- 이미지 붙여넣기 업로드 중에는 진행 상태와 실패 복구 경로를 제공합니다.
- 블록 추가 메뉴는 넓은 클릭 영역과 명확한 타입 설명을 제공합니다.
- 미니맵 노드는 현재/상위/하위 위치 인지가 되도록 공간 배치를 유지합니다.

### Don't

- 첨부/노트를 폼 블록에 흡수해 태스크 기본 기능의 일관 진입점을 깨지 않습니다.
- 레거시 파일 필드를 우회 경로로 다시 저장 가능하게 두지 않습니다.
- 미니맵을 장식 요소로만 두고 네비게이션/상태 인지 기능을 생략하지 않습니다.

## CRUD 보완 체크 (디자인/기획 공통)

- Create: FREEFORM 블록 생성은 타입 선택 즉시 생성되며 기본 편집 포커스를 제공합니다.
- Read: FREEFORM/TEMPLATED 상태에 따라 읽기 구조와 안내 문구를 분리해 혼동을 줄입니다.
- Update: 템플릿 적용/변경 시 Form Output 보강/변경 영향 범위를 사전 고지합니다.
- Delete: 블록/첨부 삭제는 복구 가능 여부와 본문 참조(토큰/프리뷰) 처리 규칙을 명시합니다.

## 엣지케이스 / 예외사항

- 이미지 붙여넣기 업로드 실패: 본문 삽입 롤백 + 실패 이유 + 재시도 동선 제공
- 대용량 이미지/연속 붙여넣기: 업로드 큐 상태 표시와 입력창 응답성 유지
- parent 변경 중 cycle 발생: 저장 차단 + 이유(자기 자신/하위 연결 불가) 설명
- 템플릿 변경 후 기존 자유 블록과 충돌: 마이그레이션/보존/폐기 정책을 UI에 명확히 노출
- 미니맵 대상 노드가 권한 변경으로 비가시 상태가 된 경우: 노드 수/연결선 표현 규칙 고정

## 범위

- 자유 노드 생성과 parent 연결
- `FREEFORM`/`TEMPLATED` 구조 상태
- 템플릿 적용과 Form Output 활성화
- `__task_description` 마크다운 리치에디터와 미리보기
- 클립보드 이미지 붙여넣기 기반 첨부 업로드/본문 참조 삽입
- FREEFORM 블록형 Form Output 구성(텍스트/긴텍스트/체크리스트/인용/숫자)
- 레거시 파일 필드(`__task_files`, `FILE`) 제거 및 저장 시 재유입 방지
- 우측 패널 상단 1-depth 관계/구조 맥락 미니맵
- 우측 패널에서 미니맵 바로 아래 의존성/전파 영향 정보 블록
- unit/folder/list 무결성
- 리스트/보드/백로그/결정 그래프 뷰 탭

## 시스템 요구사항

1. `templateType`과 `templateId`는 nullable이어야 합니다.
2. FREEFORM 태스크는 템플릿 없이 생성/수정/parent 연결이 가능해야 합니다.
3. 템플릿 적용 시 `structureState=TEMPLATED`가 되고 `formValues`는 기존값을 보존하며 필드 키 누락분을 보강합니다.
4. `unitId`, `folderId`, `listId` 조합은 생성/수정 시 항상 검증합니다.
5. parent 연결은 Work Graph cycle을 만들 수 없습니다.
6. 계층과 결정 그래프는 별도 메뉴가 아니라 태스크 뷰 탭으로 제공됩니다.
7. `__task_description` 편집은 마크다운 리치에디터를 사용해야 합니다.
8. 편집기 이미지 붙여넣기(`Ctrl/Cmd+V`)는 첨부 업로드 후 본문 참조 토큰 삽입으로 처리해야 합니다.
9. FREEFORM Form Output은 블록 단위 추가/편집 흐름을 제공해야 합니다.
10. 템플릿/태스크 저장 시 레거시 파일 필드는 항상 제거되어야 합니다.
11. 태스크 상세 우측 상단은 parent/current/children 1-depth 맥락 미니맵을 제공해야 합니다.
12. 의존성/영향 정보는 별도 탭이 아니라 미니맵 바로 아래 인접 블록으로 제공해야 합니다.
13. 우측 패널은 상단 맥락 블록(미니맵+의존성), 탭, 탭 콘텐츠가 잘림/겹침 없이 순서대로 모두 보이도록 레이아웃을 유지해야 합니다.

## 수용 기준

- 사용자가 자유 노드를 만들고 상위 노드에 연결할 수 있습니다.
- 사용자가 자기 자신 또는 descendant를 parent로 연결하려 하면 저장 전에 실패합니다.
- 반복/규칙이 필요한 노드에 템플릿을 적용해 Form Output을 활성화할 수 있습니다.
- 리스트, 보드, 백로그, 결정 그래프 탭 전환 시 같은 태스크 데이터를 다른 방식으로 볼 수 있습니다.
- 잘못된 folder/list 조합은 저장 전에 실패합니다.
- 리치에디터에서 이미지 붙여넣기 시 첨부가 생성되고 본문에 참조가 삽입됩니다.
- 자유폼 블록 추가 메뉴에서 블록 타입을 선택해 Form Output을 구성할 수 있습니다.
- 우측 미니맵에서 상/하위 노드 존재(head/tail)를 즉시 인지하고 노드 클릭 이동이 가능합니다.
- 미니맵 바로 아래에서 "나를 막는 항목/내가 막는 항목/전파 범위"를 확인하고 관련 태스크로 이동할 수 있습니다.
- 우측 패널에서 관계 섹션, 탭, 탭 콘텐츠가 여백/겹침 문제 없이 연속된 흐름으로 표시됩니다.

## 수용 기준 (User POV)

- 사용자는 FREEFORM에서 빠르게 블록을 추가해 초안을 만들고, 필요 시 템플릿 적용으로 정형화할 수 있습니다.
- 설명/노트 작성 중 이미지를 붙여넣으면 별도 업로드 화면 없이 본문에 연결됩니다.
- 태스크 상세 우측에서 이 태스크가 루트인지 리프인지 즉시 판단할 수 있습니다.
