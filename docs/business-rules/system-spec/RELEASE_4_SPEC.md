# Release 4 Spec

## 목표

운영자가 시스템을 지속적으로 개선할 수 있도록 승인 정책, 알림 설정, 분석, 관리자 기능을 안정화합니다.

## User Story (디자인 기준)

- 운영자로서, 나는 승인 정책을 조직 구조에 맞게 관리하고 싶다. 그래서 결정 품질과 책임 경로를 유지한다.
- 사용자로서, 나는 알림 채널/컴포넌트를 내 역할에 맞게 조절하고 싶다. 그래서 노이즈를 줄이고 중요한 신호를 놓치지 않는다.
- 관리자/리더로서, 나는 분석 지표를 실제 활동 기반으로 보고 싶다. 그래서 개선 우선순위를 근거 있게 잡는다.

## Design Scope (UI/상태/피드백)

- 승인정책 화면은 정책 범위(글로벌/유닛), 승인 라인, 조건을 읽기 쉬운 계층으로 보여줍니다.
- 알림 설정 화면은 채널 단위와 Inbox 컴포넌트 단위 토글을 분리해 제어 대상을 명확히 합니다.
- Inbox의 read/ack/remind/read-all은 상태 전이 피드백(즉시 반영, 실패 복구)을 제공합니다.
- 분석 화면은 지표 정의와 계산 출처(engagement event 기반)를 설명 가능한 형태로 제공합니다.
- 관리자 위험 작업(자기 자신 삭제/강등)은 경고/차단 UX를 명확히 둡니다.

## Do / Don't

### Do

- 운영 기능은 "현재 상태 -> 변경 결과"를 예측 가능하게 설계합니다.
- read-all 같은 대량 액션은 영향 범위를 명확히 안내합니다(본인 Inbox만).
- 분석 지표 카드에 용어 정의와 해석 포인트를 함께 제공합니다.

### Don't

- 위험 작업을 일반 수정 액션과 동일한 시각적 우선순위로 배치하지 않습니다.
- 알림 토글의 적용 범위(채널/컴포넌트)를 혼합 표기하지 않습니다.
- API 지표와 다른 임의 계산값을 UI에 노출하지 않습니다.

## CRUD 보완 체크 (디자인/기획 공통)

- Create: 승인정책/초대/구독 생성은 범위와 대상 검증을 통과한 경우만 완료 처리합니다.
- Read: 관리자/일반 사용자별 조회 범위를 분리하고, 비가시 항목은 명확히 차단합니다.
- Update: 정책/알림 수정은 적용 범위(글로벌/유닛/개인)를 변경 전후로 비교 가능하게 보여줍니다.
- Delete: 멤버/구독/정책 삭제는 영향 대상과 안전 제약(자기 자신 강등/삭제 불가)을 선노출합니다.

## 엣지케이스 / 예외사항

- read-all 실행 직후 새 알림 유입: 일괄 읽음 범위 시점을 UI에 명시
- 관리자 권한이 세션 중 변경된 경우: 위험 액션 즉시 비활성 + 재검증
- 승인정책 삭제 시 기존 태스크 참조가 남아있는 경우: 차단 또는 대체 정책 지정 플로우 제공
- Push subscription 중복 등록/만료 토큰: idempotent 처리와 정리 메시지 제공
- 분석 데이터 지연/집계 실패: 마지막 집계 시각과 fallback 상태를 분리 표시

## 범위

- 승인 정책 관리
- 알림 설정과 Push subscription
- Inbox 읽음/확인/리마인드 처리
- engagement event 기반 분석
- 관리자 멤버/초대/권한 관리
- 회귀 테스트 범위 확대

## 시스템 요구사항

1. 승인 정책은 글로벌/유닛 범위를 지원합니다.
2. 승인 정책 라인은 승인자, 조건, 최종 승인자 여부를 표현합니다.
3. Inbox는 read, acknowledge, remind, read-all 처리를 지원합니다.
4. 알림 설정은 채널과 Inbox 컴포넌트별 토글을 저장합니다.
5. Push subscription 등록/해제를 지원합니다.
6. `/api/analytics/retention`은 현재 데이터와 engagement event에서 지표를 계산합니다.
7. 관리자 자기 자신 삭제/강등 같은 위험 작업은 안전 제약을 둡니다.

## 수용 기준

- 관리자 화면에서 승인 정책을 만들고 수정할 수 있습니다.
- 사용자가 알림 채널과 컴포넌트별 알림 수신을 조정할 수 있습니다.
- 관리자가 read-all을 실행해도 다른 사용자의 Inbox 읽음 상태는 변경되지 않습니다.
- 분석 화면에서 Objective 1과 KR 1.1 관련 지표가 API 값과 일치합니다.
- 보안, 인가, 무결성, 멘션, 분석 회귀 테스트가 통과합니다.

## 수용 기준 (User POV)

- 운영자는 정책 수정 후 승인 라인 변화가 화면에서 즉시 확인됩니다.
- 사용자는 알림을 끈 항목에서 불필요한 알림이 줄고, 켠 항목 신호는 유지되는 것을 체감합니다.
- 관리자는 read-all 실행 시 내 Inbox만 변하는 것을 명확히 확인할 수 있습니다.
