# Release 4 시스템 스펙

## 목표

운영/분석 관점에서 "지속 개선 가능한 시스템"을 완성합니다.  
핵심은 이벤트 축적, 지표 계산, 관리 기능 신뢰성입니다.

## 범위

- 엔게이지먼트 이벤트 누적
- 리텐션/협업 분석 API 고도화
- 관리자 멤버/초대 관리 안정화
- 회귀 테스트 범위 확대(정책 변경 대응력)

## 필수 시스템 요구사항

1. 주요 사용자 행동은 `engagement`로 기록
2. 분석 API는 최신 이벤트/콘텐츠 기반 계산
3. 관리자 변경(멤버 삭제/역할 변경)은 안전 제약 준수
4. 정책 변경 시 테스트/문서 동시 갱신

## 수용 기준

- 관리자 분석 화면 지표가 API 결과와 일치한다.
- ADMIN 자기 자신 삭제/강등 금지 정책이 유지된다.
- 보안/인가/무결성 핵심 회귀 테스트가 모두 통과한다.
