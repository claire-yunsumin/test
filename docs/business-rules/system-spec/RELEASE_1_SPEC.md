# Release 1 Spec

## 목표

기본 워크스페이스가 안전하게 동작하도록 인증, 권한, 가시성, 기본 CRUD를 고정합니다.

## User Story (디자인 기준)

- 멤버로서, 나는 내가 볼 수 있는 태스크만 안전하게 조회하고 싶다. 그래서 불필요한 정보 노출 없이 내 일에 집중할 수 있다.
- 워처로서, 나는 태스크 맥락은 읽되 핵심 필드를 실수로 바꾸지 않길 원한다. 그래서 책임 경계가 명확해진다.
- 운영자로서, 나는 권한 오류가 일관된 방식으로 드러나길 원한다. 그래서 사용자 안내와 지원 대응이 쉬워진다.

## Design Scope (UI/상태/피드백)

- 인증 실패는 작업 화면 대신 명확한 오류 상태로 전환합니다.
- 권한 부족 액션은 숨기기보다 비활성 + 이유 안내를 기본으로 합니다.
- 조회 가능/수정 불가 사용자는 동일한 상세 화면 구조를 유지하되 편집 컨트롤만 잠급니다.
- 태스크/노트/댓글 CRUD 실패는 공통 에러 포맷을 사용자 메시지로 일관되게 변환합니다.

## Do / Don't

### Do

- 조회 권한과 수정 권한을 분리해 UI affordance를 설계합니다.
- 워처의 읽기 맥락(본문, 타임라인, 첨부, 노트)은 유지합니다.
- 실패 상태에서 다음 행동(닫기, 재시도, 권한 요청)을 명확히 제공합니다.

### Don't

- 권한 없는 사용자에게 편집 가능한 것처럼 보이는 입력 UI를 노출하지 않습니다.
- 같은 오류를 화면마다 다른 용어/형태로 표현하지 않습니다.
- 가시성 밖 리소스를 회색 처리로만 보여주지 않습니다(서버 차단 + 명확한 실패 처리).

## CRUD 보완 체크 (디자인/기획 공통)

- Create: 생성 액션은 권한/필수값 충족 시에만 활성화하고, 실패 시 필드 단위 이유를 보여줍니다.
- Read: 조회 가능하지만 수정 불가인 경우, 동일 화면 구조에서 read-only 상태와 이유를 유지합니다.
- Update: 수정 가능 사용자에게만 저장 affordance를 노출하고, 충돌/권한 변경 시 재검증합니다.
- Delete: 삭제 액션은 권한 사용자만 접근 가능해야 하며, 확인 단계와 영향 범위 안내를 제공합니다.

## 엣지케이스 / 예외사항

- 인증 만료 상태에서 저장 시도: 즉시 실패 후 재인증/재시도 경로 노출
- URL 직접 접근으로 가시성 밖 리소스 요청: 데이터 미노출 + 차단 피드백
- watcher가 브라우저 자동완성/단축키로 수정 요청을 보내는 경우: 서버 차단 + UI 동기화
- 삭제 직전 타 사용자가 리소스를 변경한 경우: 최신 상태 재조회 후 사용자 확인

## 범위

- 데모 인증 헤더 `X-Demo-User-Id`
- 역할 계층 `MEMBER < OWNER < ADMIN < SUPER_ADMIN`
- 태스크, 노트, 댓글 기본 CRUD
- 태스크 가시성 계산
- 공통 에러 규약

## 시스템 요구사항

1. `/health`를 제외한 API는 인증 컨텍스트를 요구합니다.
2. 사용자 식별 실패는 `401 UNAUTHORIZED`를 반환합니다.
3. 역할 미달은 `403 FORBIDDEN`을 반환합니다.
4. 관리자 역할은 전체 태스크에 접근할 수 있습니다.
5. 일반 사용자는 owner/assignee/watcher 관계와 parent chain 태스크만 볼 수 있습니다.
6. 변경성 요청은 입력 스키마 검증을 거쳐야 합니다.
7. 보이는 태스크라도 watcher 또는 parent chain만으로 접근한 사용자는 태스크 필드를 수정할 수 없습니다.

## 수용 기준

- 존재하지 않는 데모 사용자로 요청하면 인증 실패가 발생합니다.
- 일반 사용자가 가시 범위 밖 태스크를 조회/수정하면 차단됩니다.
- watcher는 태스크를 조회할 수 있지만 title, assignee, watcher, parent 같은 필드는 수정할 수 없습니다.
- 태스크, 노트, 댓글 CRUD가 공통 에러 규약을 지킵니다.
- 관리자 사용자는 전체 태스크를 조회할 수 있습니다.

## 수용 기준 (User POV)

- 워처 계정으로 상세에 들어가면 읽기는 가능하지만 저장/수정 액션은 비활성이고 이유가 보입니다.
- 권한 없는 태스크 URL에 직접 접근하면 데이터가 노출되지 않고 차단 피드백이 보입니다.
- 인증이 끊긴 상태에서 변경 시도 시 즉시 실패하고 복구 경로(재시도/재인증)가 분명합니다.
