# Release 1 시스템 스펙

## 목표

기본 워크스페이스가 "안전하게 동작"하도록 최소 기능을 고정합니다.  
핵심은 인증/인가, 가시성, 기본 CRUD입니다.

## 범위

- 데모 인증 헤더 기반 사용자 식별
- 역할 기반 접근 제어 기본 정책
- 태스크 기본 조회/생성/수정/삭제
- 노트/댓글 기본 생성/수정/삭제
- 기본 에러 규약(`UNAUTHORIZED`, `FORBIDDEN`, `VALIDATION_ERROR`)

## 필수 시스템 요구사항

1. `/health` 제외 모든 API는 인증 컨텍스트를 가져야 함
2. 권한 미달 요청은 `403 FORBIDDEN`으로 즉시 차단
3. 리소스 미존재는 `404 *_NOT_FOUND` 규약 유지
4. 변경성 요청은 Zod 입력 검증 필수

## 수용 기준

- VIEWER가 생성/수정 API 호출 시 차단된다.
- EDITOR 이상이 태스크/노트/댓글 CRUD를 수행할 수 있다.
- 잘못된 입력이 `VALIDATION_ERROR`로 일관되게 응답된다.
