# Release 1 Spec

## 목표

기본 워크스페이스가 안전하게 동작하도록 인증, 권한, 가시성, 기본 CRUD를 고정합니다.

## 범위

- 데모 인증 헤더 `X-Demo-User-Id`
- 역할 계층 `MEMBER < OWNER < ADMIN < SUPER_ADMIN`
- 태스크, 노트, 댓글 기본 CRUD
- 태스크 가시성 계산
- 공통 에러 규약

## 시스템 요구사항

1. `/health`를 제외한 API는 인증 컨텍스트를 요구합니다.
2. 사용자 식별 실패는 `401 UNAUTHORIZED`를 반환합니다.
3. 역할 미달은 `403 FORBIDDEN`을 반환합니다.
4. 관리자 역할은 전체 태스크에 접근할 수 있습니다.
5. 일반 사용자는 owner/assignee/watcher 관계와 parent chain 태스크만 볼 수 있습니다.
6. 변경성 요청은 입력 스키마 검증을 거쳐야 합니다.
7. 보이는 태스크라도 watcher 또는 parent chain만으로 접근한 사용자는 태스크 필드를 수정할 수 없습니다.

## 수용 기준

- 존재하지 않는 데모 사용자로 요청하면 인증 실패가 발생합니다.
- 일반 사용자가 가시 범위 밖 태스크를 조회/수정하면 차단됩니다.
- watcher는 태스크를 조회할 수 있지만 title, assignee, watcher, parent 같은 필드는 수정할 수 없습니다.
- 태스크, 노트, 댓글 CRUD가 공통 에러 규약을 지킵니다.
- 관리자 사용자는 전체 태스크를 조회할 수 있습니다.
