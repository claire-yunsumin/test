# KR 1.1 부속 — 자유폼/템플릿 양방향 사용자 플로우 매트릭스

## 목적

| 항목 | 내용 |
| --- | --- |
| 문서 성격 | `KR1_1_FEATURE_SPEC.md` 하위 사용자 플로우 문서 |
| 기준 관점 | 사용자는 먼저 형상화(FREEFORM)하고, 필요 시 템플릿을 적용/저장/교체한다 |
| 참고 모델 | ClickUp류의 유연한 시작 -> 점진적 구조화 흐름 |

## 사용자 플로우 커버리지

| 사용자 시나리오 | 현재 커버 여부 | 현재 처리 방식 (AS-IS) | 비고 |
| --- | --- | --- | --- |
| 1. 자유롭게 태스크를 추가하고 그대로 사용(FREEFORM 유지) | Covered | `structureState=FREEFORM`으로 생성/수정 가능, 상태/협업 데이터 동일 모델에서 동작 | KR1.1 형상화 기본 경로 |
| 2. 작업 중 템플릿을 선택해 구성폼 + 워크플로우 규칙으로 전환 | Covered | `templateId` 선택 시 `TEMPLATED` 전환, `formValues` 키 보강, 상태는 카테고리 기반 안전 매핑 | KR11-TP-v1 반영 |
| 3. 자유폼으로 해둔 내용을 새 템플릿으로 저장 후 즉시 적용 | Covered | Task Detail의 `saveAsTemplate`로 템플릿 생성 -> 해당 템플릿을 현재 태스크에 적용 | 필드 타입은 기본 `TEXT` 중심 생성 |
| 4. 기존 템플릿 중 하나를 선택해 업데이트(교체) | Covered | `templateId` 재선택으로 템플릿 교체, 폼값은 유지+신규필드 보강, 상태/정책 재검증 수행 | 전환 이벤트 추적 가능 |
| 5. 템플릿 해제 후 자유폼으로 복귀 | Covered | `templateId=null` 패치 시 `FREEFORM` 복귀, 기존 협업/이력 데이터 유지, 상태 legacy 정렬 | 전환 손실 최소화 |

| 첨부/노트 | 유지 | 유지 | 유지 | 유지 |

## FREEFORM 폼 구성 (포함 요소)

| 구분 | FREEFORM에서 포함되는 구성 | 저장/처리 방식 | 템플릿 전환 시 동작 |
| --- | --- | --- | --- |
| 기본 태스크 필드 | 제목, 담당/참여자, 일정, 상위/하위 관계 등 시스템 필드 | Task 기본 필드로 저장 | 그대로 유지 |
| 자유 폼 블록 | **기본 리치에디터 블록 1개가 디폴트로 포함**되고, 사용자가 블록을 추가/편집하는 텍스트 중심 구성(노션 스타일 블록 편집) | `formValues` 기반으로 키-값 저장 | 기존값 유지 + 템플릿 필드 누락분 보강 |
| 근거 연결 | 노트 참조(`#`), 멘션(`@`)로 연결된 논의 맥락 | Comment/Note/Timeline에 `taskId` 귀속 저장 | 그대로 유지 |
| 첨부 파일/링크 | 파일 업로드, 링크 첨부 | Attachment 리소스로 분리 저장 | 그대로 유지 |
| 워크플로우 상태 표현 | legacy 상태(`currentState`) + `workflowStatusId` | 상태 필드에 저장 | 안전 매핑(카테고리->default->legacy fallback) 적용 |

원칙:
- FREEFORM은 고정 템플릿 스키마가 없는 시작 모드이며, 사용자가 필요한 필드를 점진적으로 형성한다.
- FREEFORM 시작 시점에는 설명 작성용 기본 리치에디터 블록 1개가 포함된다.
- 템플릿 적용 시 FREEFORM 데이터는 삭제보다 보존을 우선하며, 새 템플릿 스키마에 맞게 보강/정렬한다.

  - 정형화 시점 추천: "지금 구조화를 하면 반복 비용이 감소"

### AI Ready 중복 해소 전략 (템플릿 목적/구성)

| 항목 | 중복 리스크 | 권장 대응 |
| --- | --- | --- |
| 목적 중복 (`왜 쓰는 템플릿인가`) | 이름만 다르고 목적이 같은 템플릿이 늘어나 추천 품질 저하 | 템플릿에 `purposeTag`(예: 회고/실험/요구정의)와 `successOutcome` 메타를 필수화하고, 생성 전 유사 목적 템플릿 후보를 먼저 제시 |
| 구성 중복 (`무엇으로 구성되는가`) | 필드 키/상태셋이 거의 같은 템플릿 파편화 | 저장 시 구조 fingerprint(필드 key/type + workflow status/transition)를 계산해 유사도 임계치 초과 시 "기존 템플릿 업데이트" 우선 제안 |
| 버전 분기 남발 | 사소한 수정마다 새 템플릿 생성되어 lineage 단절 | 신규 생성 대신 `기존 템플릿 버전업` 플로우를 기본값으로 두고, 생성은 예외 케이스로 제한 |
| 추천 충돌 | 같은 맥락에 상충 템플릿 다중 추천으로 의사결정 피로 발생 | top-1 기본 + 대안 1개만 노출하고, 추천 근거(목적 적합도/구성 적합도)를 분리 표시 |
| 정리 부재 | 미사용/중복 템플릿이 누적되어 모델 학습 오염 | 30/60일 미사용 템플릿 아카이브 후보 자동 표시 + 소유자 리뷰 주기 운영 |

중복 판정 기준(권장):
- 목적 중복: `purposeTag` 동일 + 최근 사용 시나리오(태스크 타입/단위) 유사도 높음
- 구성 중복: 필드/워크플로우 fingerprint 유사도 0.8 이상
- 병합 우선순위: 사용량 높은 템플릿을 기준본으로 두고 하위 변형을 버전으로 흡수

| --- | --- | --- |
| 추천 설명 가능성 | 추천 이유가 불명확하면 신뢰 하락 | "추천 근거 3줄" UI(반복 필드, 유사 템플릿, 최근 전이 패턴) 제공 |
| 사용자 통제감 | 추천이 강제처럼 보이면 반발 발생 | 수락/거절/나중에 보기/다시 추천 안함 액션 분리 |
| 편집 가능성 | 생성 템플릿 초안을 바로 수정 못하면 실무 적합도 낮음 | AI 초안 -> 사용자 편집 -> 저장의 3단계 흐름 고정 |
| 맥락 단절 | 추천이 현재 작업 맥락과 분리되면 클릭률 저하 | Task Detail 우측 패널(현재 전환 영역)에 인라인 추천 노출 |
| 실패 경험 누적 | 저품질 추천 반복 시 기능 전체 비활성화로 이어질 수 있음 | 추천 빈도 캡, 신뢰도 임계치 이하 미노출, 피드백 반영 주기 단축 |

### AI 운영 지표 제안

| 지표 | 정의 | 목적 |
| --- | --- | --- |
| Template Recommendation CTR | 추천 노출 대비 클릭 비율 | 추천 주제 적합성 측정 |
| Recommendation Acceptance Rate | 클릭 대비 실제 적용/저장 비율 | 실행 가능한 추천 품질 측정 |
| Post-Recommendation Revert Rate | 추천 적용 후 일정 시간 내 되돌림 비율 | 잘못된 추천 탐지 |
| Auto-Structured Throughput | FREEFORM -> TEMPLATED 전환 중 AI 보조 비중 | 정형화 보조 실효성 측정 |
| Evidence-linked Recommendation Rate | 추천 중 멘션/노트 근거를 포함한 비율 | 설명 가능한 추천 비율 관리 |

### 단계적 도입 권장안

| 단계 | 범위 | 게이트 |
| --- | --- | --- |
| Phase 1 | 템플릿 선택 추천(읽기 전용) | 추천 이유 표시 + 수락/거절 로그 수집 |
| Phase 2 | 템플릿 생성 초안 추천 | 사용자 편집 필수 + 저장 전 diff 확인 |
| Phase 3 | 정형화 시점 추천 자동화 | 역할 기반 opt-in + 되돌리기율 임계치 통과 시 확장 |

## 템플릿 과다 증가 대비 운영 가드레일

| 항목 | 과다 증가 리스크 | 운영 대응 |
| --- | --- | --- |
| 생성 진입 조건 | 누구나 즉시 생성 가능하면 저품질 템플릿 급증 | 기본은 "초안 템플릿"으로 생성하고, 팀 공유 템플릿은 소유자/리드 승인 후 승격 |
| 노출 제한 | 템플릿 목록 과밀로 선택 비용 증가 | 최근 사용/팀 표준/AI 추천 우선 노출, 나머지는 접기(검색 기반) |
| 수명주기 관리 | 오래된 템플릿 방치로 학습/추천 오염 | `Draft -> Active -> Deprecated -> Archived` 상태 모델 도입 |
| 자동 정리 | 미사용 템플릿 누적 | 30/60/90일 미사용 템플릿 자동 후보화 + 일괄 아카이브 |
| 중복 방지 | 유사 템플릿 반복 생성 | 생성 시 유사도 검사 후 "신규 생성" 대신 "기존 템플릿 복제/버전업" 유도 |
| 품질 하한선 | 제목만 있고 규칙 없는 템플릿 대량 생성 | 최소 요구사항(목적, 핵심 필드 N개, 기본 workflow) 미충족 시 저장 제한 |

권장 운영 정책:
- 개인 템플릿은 자유 생성, 팀 전역 템플릿은 승인 기반으로 제한한다.
- 추천/검색 기본 대상은 `Active`만 포함하고 `Deprecated/Archived`는 기본 숨김한다.
- 월 1회 템플릿 정리 리포트(신규/중복/미사용/아카이브)를 운영 지표로 점검한다.

## 노출 과밀 대응 (템플릿 셀렉터 / 템플릿 센터)

### 1) 태스크 상세 템플릿 셀렉터(즉시 적용 영역)

| 규칙 | 권장값 | 목적 |
| --- | --- | --- |
| 기본 노출 개수 | 상위 8개(최근 사용 4 + 팀 표준 2 + AI 추천 2) | 드롭다운 과밀 방지 |
| 추가 탐색 방식 | "더 보기" + 검색 입력 | 긴 목록 스크롤 제거 |
| 검색 기본 정렬 | 정확 매치 > 최근 사용 > 동일 `templateType` > 팀 표준 | 선택 속도 향상 |
| 표시 배지 | `표준`, `최근`, `추천`, `개인` | 선택 판단 비용 축소 |
| 숨김 대상 | `Deprecated/Archived` 기본 숨김 | 저품질/구버전 노출 차단 |

셀렉터 액션 원칙:
- 드롭다운은 "빠른 선택"만 담당하고, 대규모 탐색은 템플릿 센터로 분리한다.
- 검색 결과가 많으면 `templateType`과 `purposeTag` 2단 필터를 먼저 적용한다.

### 2) 템플릿 센터(관리/거버넌스 영역)

| 영역 | 노출 정책 | 운영 포인트 |
| --- | --- | --- |
| 기본 탭 | `Active` 우선, `Draft/Deprecated/Archived` 분리 탭 | 상태별 정리 동선 확보 |
| 페이지네이션 | 20개 단위 + 무한스크롤 대신 페이지 이동 | 성능/가시성 균형 |
| 정렬 프리셋 | 최근 사용, 사용량, 최신 수정, 표준 우선 | 목적별 탐색 단축 |
| 중복 관리 | 유사도 높은 템플릿 클러스터 뷰 | 병합/버전업 액션 유도 |
| 대량 정리 | 미사용/중복 후보 일괄 아카이브 | 운영비용 절감 |

센터 관리 원칙:
- "신규 생성"보다 "기존 템플릿 버전업" CTA를 상단에 우선 배치한다.
- `Active` 수가 임계치(예: 팀당 60개) 초과 시 정리 배너를 강제 노출한다.

### 3) 운영 임계치(가이드)

| 지표 | 경고 임계치 | 조치 |
| --- | --- | --- |
| 팀별 Active 템플릿 수 | 60+ | 표준 템플릿 재지정 + 미사용 정리 캠페인 |
| 셀렉터 검색 없는 선택률 | 30% 미만 | 기본 노출 구성을 재튜닝(최근/추천 비중 조정) |
| 템플릿 선택 후 즉시 변경률 | 20% 초과 | 추천 품질/라벨 명확성 점검 |
| Deprecated 비율 | 25% 초과 | 버전 병합/아카이브 배치 작업 실행 |
