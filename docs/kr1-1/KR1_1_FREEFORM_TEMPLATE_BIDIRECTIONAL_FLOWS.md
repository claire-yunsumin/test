# KR 1.1 부속 — 자유폼/템플릿 양방향 사용자 플로우 매트릭스

## 목적

| 항목 | 내용 |
| --- | --- |
| 문서 성격 | `KR1_1_FEATURE_SPEC.md` 하위 사용자 플로우 문서 |
| 기준 관점 | 사용자는 먼저 형상화(FREEFORM)하고, 필요 시 템플릿을 적용/저장/교체한다 |
| 참고 모델 | ClickUp류의 유연한 시작 -> 점진적 구조화 흐름 |

## 사용자 플로우 커버리지

| 사용자 시나리오 | 현재 커버 여부 | 현재 처리 방식 (AS-IS) | 비고 |
| --- | --- | --- | --- |
| 1. 자유롭게 태스크를 추가하고 그대로 사용(FREEFORM 유지) | Covered | `structureState=FREEFORM`으로 생성/수정 가능, 상태/협업 데이터 동일 모델에서 동작 | KR1.1 형상화 기본 경로 |
| 2. 작업 중 템플릿을 선택해 구성폼 + 워크플로우 규칙으로 전환 | Covered | `templateId` 선택 시 `TEMPLATED` 전환, `formValues` 키 보강, 상태는 카테고리 기반 안전 매핑 | KR11-TP-v1 반영 |
| 3. 자유폼으로 해둔 내용을 새 템플릿으로 저장 후 즉시 적용 | Covered | Task Detail의 `saveAsTemplate`로 템플릿 생성 -> 해당 템플릿을 현재 태스크에 적용 | 필드 타입은 기본 `TEXT` 중심 생성 |
| 4. 기존 템플릿 중 하나를 선택해 업데이트(교체) | Covered | `templateId` 재선택으로 템플릿 교체, 폼값은 유지+신규필드 보강, 상태/정책 재검증 수행 | 전환 이벤트 추적 가능 |
| 5. 템플릿 해제 후 자유폼으로 복귀 | Covered | `templateId=null` 패치 시 `FREEFORM` 복귀, 기존 협업/이력 데이터 유지, 상태 legacy 정렬 | 전환 손실 최소화 |

## 데이터 구성요소별 양방향 처리

| 구성요소 | FREEFORM 유지 | FREEFORM -> TEMPLATED | TEMPLATED -> FREEFORM | 템플릿 교체(A->B) |
| --- | --- | --- | --- | --- |
| 구조 모드 | `FREEFORM` | `TEMPLATED` 전환 | `FREEFORM` 복귀 | `TEMPLATED` 유지 |
| Form 구성 | 자유폼 블록/필드 편집 | 기존값 유지 + 템플릿 필드 보강 | 기존값 유지 | 기존값 유지 + 신규필드 보강 |
| 워크플로우 상태 | legacy 상태 체계 사용 | 카테고리 기반 안전 매핑 | legacy 상태 정렬 | 카테고리 기반 안전 매핑 |
| 승인정책 | 기존 정책 사용/없음 | 자동 재선정 없음 + 유효성 재검증 | 재검토 플래그 해제 | 자동 재선정 없음 + 유효성 재검증 |
| 스레드/타임라인 | 유지/누적 | 유지/누적 | 유지/누적 | 유지/누적 |
| 첨부/노트 | 유지 | 유지 | 유지 | 유지 |

## 제품 관점 해석

| 관점 | 해석 |
| --- | --- |
| 장점 | 사용자가 자유롭게 시작하고 필요할 때 구조화 가능(마찰 낮음) |
| 장점 | 전환 시 스레드/노트/타임라인/첨부를 잃지 않아 작업 연속성 높음 |
| 한계 | 템플릿 품질이 낮으면 fallback/수동 보정 비율이 높아질 수 있음 |
| 운영 필요 | 매핑 성공률/수동 보정률을 기준으로 템플릿 상태 설계 품질을 개선해야 함 |
